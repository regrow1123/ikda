# 읽다 (ikda) — Product Requirements Document

## 1. 개요

**읽다**는 책 버전 왓챠다. 읽은 책에 평점과 리뷰를 남기고, 다른 독서가들의 리뷰를 탐색하며, 소셜 기능을 통해 취향이 맞는 사람들과 연결되는 서비스.

- **플랫폼:** iOS, Android, Web (Flutter)
- **백엔드:** Supabase (Auth, Database, Storage, Edge Functions)
- **책 데이터:** 알라딘 API (메인) + 네이버 책 검색 API (보조)

---

## 2. 타겟 유저

- 한국어 도서를 읽는 독서가
- 읽은 책을 기록하고 싶은 사람
- 취향 기반 책 추천을 받고 싶은 사람 (Phase 2)

---

## 3. MVP 기능 (Phase 1)

### 3.1 인증
- 이메일/비밀번호 회원가입 & 로그인
- 소셜 로그인 (Google, Apple)
- 프로필 설정 (닉네임, 프로필 이미지, 자기소개)

### 3.2 책 검색 & 상세
- 알라딘 API 기반 책 검색
- 책 상세 정보 (표지, 제목, 저자, 출판사, 출간일, 설명)
- 해당 책의 평점 통계 & 리뷰 목록

### 3.3 평점 & 리뷰
- 별점 (0.5~5.0, 0.5 단위)
- 텍스트 리뷰 작성/수정/삭제
- 리뷰에 좋아요
- 스포일러 토글
- 읽은 날짜 기록

### 3.4 내 책장
- 읽은 책 / 읽고 있는 책 / 읽고 싶은 책 상태 관리
- 상태별 필터링 & 정렬 (평점순, 최신순)
- 독서 통계 (월별/연별 읽은 권수, 평균 평점)

### 3.5 소셜
- 유저 팔로우/언팔로우
- 피드: 팔로우한 유저의 리뷰 & 평점 활동
- 유저 프로필 페이지 (책장 공개, 리뷰 목록)
- 유저 검색

### 3.6 탐색
- 베스트셀러 / 신간 목록 (알라딘 API)
- 최근 인기 리뷰
- 카테고리별 탐색

---

## 4. Phase 2 (추후)

- **추천 알고리즘:** 협업 필터링 기반 — 비슷한 평점 패턴의 유저가 좋아한 책 추천
- **독서 모임:** 그룹 생성, 같이 읽기
- **댓글:** 리뷰에 댓글 달기
- **알림:** 팔로우 유저 새 리뷰, 좋아요 등
- **공유:** SNS 공유 카드

---

## 5. 비기능 요구사항

- **성능:** 검색 결과 2초 이내, 피드 로딩 1초 이내
- **보안:** Supabase RLS로 데이터 접근 제어
- **확장성:** 유저 10만 기준 설계
- **오프라인:** 내 책장은 로컬 캐싱
- **접근성:** 기본 접근성 가이드라인 준수

---

## 6. 성공 지표

- DAU/MAU 비율
- 유저당 평균 리뷰 수
- 리텐션 (7일, 30일)
- 팔로우 관계 수 (소셜 활성도)
